# Writer Skill Core Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Milestone 3: project-installable Skills that can be imported, browsed, selected for Agent Sessions, exposed to the model through bounded context/tooling, and clearly marked when bundled scripts are present but disabled.

**Architecture:** Add a focused `skill` backend package for persistence-backed import, parsing, browsing, and selection. Agent Core integrates with Skill Core through an optional `SkillService` dependency, a new model-facing `skill` context tool, prompt context source snapshots for available/selected skill guidance, and `skill_id` propagation into structured writes, candidates, revisions, prompt records, and activity events. The React app adds a Skill browser/settings surface and skill selectors in chat/quick actions without executing arbitrary bundled scripts.

**Tech Stack:** Go 1.26, chi, sqlc, goose, SQLite, existing Agent Core provider/tool loop, React, TypeScript, Bun, Vite.

---

## Scope

This plan implements Milestone 3 only:

- Skill import/install from uploaded zip archives and server-local directories.
- Skill browsing for installed project skills.
- Parsing `SKILL.md` frontmatter with `name`, `description`, optional routing metadata, and script-disabled metadata.
- Importing skill assets: instructions, templates, references, data files, and scripts as inert stored files.
- Explicit script-disabled handling in backend responses, prompt guidance, tool output, and UI.
- Manual skill selection for Agent Sessions and quick actions.
- Optional routing hints that let the frontend and prompt assembly recommend skills by action/content kind without automatic execution.
- A model-facing `skill` tool that loads one installed skill's instructions/assets on demand with bounded model-visible output and full tool-result artifacts.
- Prompt context source snapshots for available skills and selected skill guidance.
- Activity trail and prompt record metadata that identify selected/loaded skills.
- Agent-created revisions already have `skill_id`; this plan wires it from selection through structured writes.

This plan does not implement:

- Executing skill scripts.
- Admin approval for script execution.
- Remote marketplace/community discovery.
- Live folder sync for installed skills.
- Automatic LLM-based skill routing before a provider turn.
- Multi-author skill permissions.

## Source Decisions

The product spec and ADRs require importing skill instructions, routing metadata, templates, and data files while disabling scripts. Writer follows the useful OpenCode pattern: baseline prompt context lists available/selected skill summaries, and a model-facing `skill` tool loads full skill content only when needed. This preserves Writer's "whole-project context is tool-accessible" rule and avoids putting every installed skill file into every provider request.

## Data Model

Use server-owned IDs with the existing timestamp/counter style in service code. Store imported file bytes as text because v1 skill assets are Markdown/text instructions, templates, references, data, and script source; reject files that are not valid UTF-8.

Skill names are unique per project. A re-import with the same normalized name updates the existing skill and replaces its files/routing hints in one transaction.

Script files are imported as files with `purpose = 'script'`, but `execution_enabled = 0` always. They are visible to author and model output only as disabled files; the backend never executes them.

Routing metadata lives in `skill_routing_hints` so later matching can be SQL-backed. The parser accepts routing frontmatter in either of these forms:

```yaml
route:
  actionKinds: [rewrite, read_check]
  contentKinds: [chapter, story_bible_entry]
  tags: [style, continuity]
  priority: 50
```

```yaml
routing:
  actions: [continuation]
  content: [chapter]
  tags: [drafting]
```

The service normalizes both forms to `action_kind`, `content_kind`, `tag`, and `priority`.

```go
func TestParseSkillArchiveRejectsUnsafePaths(t *testing.T) {

func writeZipFile(t *testing.T, zw *zip.Writer, name string, body string) {
	t.Helper()
	w, err := zw.Create(name)
	if err != nil {
		t.Fatalf("create zip entry %s: %v", name, err)
	}
	if _, err := w.Write([]byte(body)); err != nil {
		t.Fatalf("write zip entry %s: %v", name, err)
	}
}

func assertImportedFile(t *testing.T, files []ImportedSkillFile, path string, purpose FilePurpose, disabled bool) {
	t.Helper()
	for _, file := range files {
		if file.RelativePath == path {
			if file.Purpose != purpose || file.ScriptDisabled != disabled {
				t.Fatalf("%s = (%s, %v), want (%s, %v)", path, file.Purpose, file.ScriptDisabled, purpose, disabled)
			}
			return
		}
	}
	t.Fatalf("missing imported file %s", path)
}
```

```go
package skill

import (
	"archive/zip"
	"bytes"
	"encoding/json"
	"errors"
	"fmt"
	"io"
	"path"
	"sort"
	"strings"
	"unicode/utf8"
)

const maxSkillArchiveBytes int64 = 5 << 20

func ParseSkillArchive(reader io.ReaderAt, size int64, sourceLabel string) (ImportedSkill, error) {
	if size > maxSkillArchiveBytes {
		return ImportedSkill{}, fmt.Errorf("skill archive is %d bytes, max %d", size, maxSkillArchiveBytes)
	}
	zr, err := zip.NewReader(reader, size)
	if err != nil {
		return ImportedSkill{}, fmt.Errorf("open skill zip: %w", err)
	}

	type entry struct {
		path string
		body string
	}
	entries := make([]entry, 0, len(zr.File))
	for _, file := range zr.File {
		if file.FileInfo().IsDir() {
			continue
		}
		relative, err := safeRelativePath(file.Name)
		if err != nil {
			return ImportedSkill{}, err
		}
		body, err := readZipText(file)
		if err != nil {
			return ImportedSkill{}, err
		}
		entries = append(entries, entry{path: relative, body: body})
	}
	sort.Slice(entries, func(i, j int) bool { return entries[i].path < entries[j].path })

	root := commonSkillRoot(entries)
	for index := range entries {
		entries[index].path = strings.TrimPrefix(entries[index].path, root)
	}

	var skillFile *entry
	for index := range entries {
		if entries[index].path == "SKILL.md" {
			skillFile = &entries[index]
			break
		}
	}
	if skillFile == nil {
		return ImportedSkill{}, errors.New("skill archive must contain SKILL.md")
	}

	frontmatter, instructions, err := parseSkillMarkdown(skillFile.body)
	if err != nil {
		return ImportedSkill{}, err
	}
	name := normalizeSkillName(frontmatter.Name)
	if name == "" {
		return ImportedSkill{}, errors.New("skill frontmatter name is required")
	}

	imported := ImportedSkill{
		Name:                 name,
		DisplayName:          frontmatter.Name,
		Description:          frontmatter.Description,
		InstructionsMarkdown: strings.TrimSpace(instructions),
		MetadataJSON:         frontmatterMetadataJSON(frontmatter),
		SourceLabel:          sourceLabel,
		ScriptsDisabled:      true,
		RoutingHints:         frontmatter.routingHints(),
	}
	for _, item := range entries {
		purpose, disabled := classifySkillFile(item.path)
		file := ImportedSkillFile{
			RelativePath:   item.path,
			Purpose:        purpose,
			MediaType:      "text/plain; charset=utf-8",
			BodyText:       item.body,
			Bytes:          int64(len([]byte(item.body))),
			ScriptDisabled: disabled,
		}
		if purpose == FilePurposeScript {
			imported.ScriptCount++
		}
		imported.Files = append(imported.Files, file)
	}
```

```go
func (s *Service) Install(ctx context.Context, input InstallInput) (Skill, error) {
	if strings.TrimSpace(input.ProjectID) == "" || strings.TrimSpace(input.Imported.Name) == "" {
		return Skill{}, ErrInvalidInput

	now := nowString()

	err := s.inTx(ctx, func(q *store.Queries) error {
		if _, err := q.GetStoryProjectByID(ctx, input.ProjectID); err != nil {

		}); err == nil {
			skillID = existing.ID
		} else if !errors.Is(err, sql.ErrNoRows) {
			return fmt.Errorf("get existing skill: %w", err)
		}

		if err := q.UpsertSkill(ctx, store.UpsertSkillParams{
			ID:                   skillID,

			Description:          input.Imported.Description,

			SourceType:           string(input.SourceType),
			SourceLabel:          emptyDefault(input.SourceLabel, input.Imported.SourceLabel),
			ScriptCount:          input.Imported.ScriptCount,
			ScriptsDisabled:      boolInt(input.Imported.ScriptsDisabled),
			MetadataJson:         defaultJSON(input.Imported.MetadataJSON),
			InstalledAt:          now,
			UpdatedAt:            now,
		}); err != nil {
			return fmt.Errorf("upsert skill: %w", err)
		}
		if err := q.DeleteSkillFiles(ctx, skillID); err != nil {
			return fmt.Errorf("delete old skill files: %w", err)
		}
		if err := q.DeleteSkillRoutingHints(ctx, skillID); err != nil {
			return fmt.Errorf("delete old routing hints: %w", err)
		}
		for _, file := range input.Imported.Files {
			if err := q.CreateSkillFile(ctx, store.CreateSkillFileParams{
				ID:             newID("skillfile"),
				SkillID:        skillID,
				RelativePath:   file.RelativePath,

		return Skill{}, err
```

```ts
  return requestJSON<WriterSkill>(projectSkillPath(projectId, `skills/${encodeURIComponent(skillId)}`), { signal });
}

export function importSkill(projectId: string, file: File): Promise<WriterSkill> {
  return requestJSON<WriterSkill>(projectSkillPath(projectId, "skills/import"), {
    method: "POST",
    body: file,
    headers: { "Content-Type": "application/zip" },
  });
}

export function listSessionSkills(projectId: string, sessionId: string, signal?: AbortSignal): Promise<WriterSkill[]> {
  return requestJSON<WriterSkill[]>(
    `/api/projects/${encodeURIComponent(projectId)}/agent/sessions/${encodeURIComponent(sessionId)}/skills`,
    { signal },
  );
}

export function selectSessionSkills(projectId: string, sessionId: string, skillIds: string[]): Promise<WriterSkill[]> {
  return requestJSON<WriterSkill[]>(
    `/api/projects/${encodeURIComponent(projectId)}/agent/sessions/${encodeURIComponent(sessionId)}/skills`,
    {
      method: "PUT",
      body: JSON.stringify({ skillIds }),
      headers: { "Content-Type": "application/json" },
    },
  );
}
```
