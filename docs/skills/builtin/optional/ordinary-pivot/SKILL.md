# $ordinary-pivot

Story design for authors who want protagonists pulled into larger conflict through ordinary work, access, obligations, or systemic position.

## Use When

- The main character should matter because of structural access, not chosen-one logic.
- A mundane role needs to reveal a bigger system.
- The author wants inevitable involvement rather than coincidence.

## Do Not Use When

- The story is already built around heroic destiny or overt exceptionalism.
- The request is mainly about team dynamics; `$underdog-team` may fit better.
- The character has no meaningful system around them to pivot through.

## Writer Workflow

1. Read the target Chapter, Story Text, and any world notes shaping the system.
2. Identify the protagonist's ordinary role and the access it naturally grants.
3. Design the reveal, lock-in pressure, and competing interests around that role.
4. Check that the involvement feels structurally inevitable in this world.
5. Draft or revise only when the author explicitly asks for applied story work.

## Writer Output Handling

- Return the position map and conflict design in chat by default.
- Create an Attached Note when the work belongs to one Chapter or one character problem.
- Create or update a Project Note when the author wants a reusable concept or arc plan.
- Propose Story Bible updates only if the role or system becomes durable canon the author wants tracked.
- Use Structured Writes only when the author explicitly asks to draft or rewrite selected Story Text.

## Script Compatibility

This source skill has no required helper scripts. The Milestone 3.5 rewrite works through Writer-native concept design and revision guidance.
